Purpose
The Victorian Department of Health and Department of Families, Fairness and Housing, from here on referred
to as the departments, engage third parties for the delivery of ICT services as well as Client/Community
services to the organisation.
This standard has been written to outline the expectations the departments have of all third parties that are
engaged to work with the departments’ IT, information, and data. Applying the controls set out in this standard
will provide assurance to the departments that third parties understand the importance of securing systems
and information and that they will work with the department to ensure the confidentiality, integrity and
availability of departmental information.

Authority
This standard derives its’ authority from the Information Security Policy which has been approved by the
departments’ executive committees. The departments’ Information Security Policy requires that the following
are adhered to:
           We will ensure that third parties securely collect, hold, manage, use, disclose or transfer public
           sector information.
           Sharing of the department’s information with external parties will be based on an understanding of
           the recipient’s capability to adequately protect the information.
With some contracts / agreements, specified controls may have already been included. The following controls
should be applied in conjunction with any information security requirements described in the contract /
agreement, and where there are similar controls, those that are stipulated in the contract / agreement will take
precedence over the controls in this document. If there are no controls stipulated in the contract / agreement,
then the controls described in the standard shall apply.

Scope
This standard applies to all third parties, and their staff, that have been engaged by the departments to:
•     work with departmental information / data
•     collect information / data on the departments’ behalf
•     provide or develop systems / applications, either on-premise or cloud-based, IT Services, to the
      departments

requirements.
The specific requirement for each topic area is provided with an associated rationale for the requirement.

applicability of the control and any specific references.

Appendix 1 - Baseline Technical Controls
The table below describes the technical control and its description. It is intended to provide third-parties with an overview of the expected security controls
that they should implement (if not already done so) to enable them to secure their networks, systems / applications and data / information. This in turn will
provide assurance to the department that the third party will have the minimum controls to keep departmental data / information secure.

 Control name                         Control description
 Multi-Factor Authentication          Multi-factor authentication is defined as ‘a method of authentication that uses two or more authentication
 (MFA)                                factors to authenticate a single claimant to a single authentication verifier’.
                                      MFA is a core component of a strong identity and access management (IAM) policy. Rather than just
                                      asking for a username and password, a person trying to access a system or network must provide
                                      additional identification verifiers.
                                      The authentication factors that make up a multi-factor authentication request must come from two or more
                                      of the following:
                                         1. something the claimant knows (e.g. a personal identification number (PIN), password or response
                                            to a challenge)
                                         2. something the claimant has (e.g. a physical token, smartcard, mobile apps such as authenticators
                                            or software certificate)
                                         3. something the claimant is (biometrics e.g. a fingerprint or iris scan).
 Application control                  Application control is a security approach designed to protect against malicious code (also known as
                                      malware) executing on systems. When implemented robustly, it ensures only approved applications (e.g.
                                      executables, software libraries, scripts, installers, compiled HTML, HTML applications, control panel
                                      applets and drivers) can be executed.
                                      While application control is primarily designed to prevent the execution and spread of malicious code, it
                                      can also prevent the installation or use of unapproved applications.
 Restrict administrative privileges   Users with administrative privileges for operating systems and applications can make significant changes
                                      to their configuration and operation, bypass critical security settings and access sensitive information.
