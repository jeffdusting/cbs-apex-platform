Please find Downers response to your information request:

*Full list of data as mentioned in Q2



| RFI# | Date | CBS Question | Downer Answer |

| 001 | 26MAY | Any baseline data on energy use in the past 12 - 24 months? | HVAC Hardware
Downer is currently conducting a trial of the HVAC Hardware energy efficiency on B32. The trial commenced in Dec-20 and will be completed in Sep-21 (9 months covering both the summer and winter periods). This trial collects reference data of un-modified units on the same Set as the modified units and will provide a baseline position.
 
 
HVAC Software
There is no baseline yet for the HVAC Software. A one (1) month trial will commence later this month on B39 for each of Duct Heater and Setback Mode software changes.
 
Existing on train power meters.
Each A and B Set has onboard energy meters which track the HVAC energy consumption on a half Set basis. The data from these meters is sent from Set to Shore every 10 minutes and then stored in the Databank, so yes we can build up a history of how much energy the HVAC system has used on each train over a lengthy period.
 
There will be some missing data (some Sets don’t have energy meters that work correctly) and it will take some time to get the data from TrainDNA/Databank, however it is possible to build up a history of how much energy was used by the HVAC system by each Set.
 
The problem with this is what is hinted at in the next question, different climate, passenger loading and operational usage of the Sets will impact how much energy is used, so several assumptions would need to be made and potentially data adjusted to account for climate and passenger loading. Doing this is even more work to download different data sets and evaluate them. |

| 002 | 26MAY | A list of available data points that might impact the electricity consumption of a Train Set; passenger load, routes/ schedules, HVAC settings, etc, etc. Also the measurement intervals for each data type; constant, hourly, daily, etc. | There are several factors which will influence the performance of the energy consumption of the HVAC unit.
Climate (i.e. Fresh Air Temp, Solar Load, Humidity etc.)
Operational usage (i.e. where in Sydney the Set runs which leads to different climates experienced by each Set, different amounts of stabling experienced, how often they left with doors open etc.)
Patronage
Driver/Guard control of HVAC in Crew Cab
Etc.
 
Each HVAC on a train (16 in total) records its operational in a log, the log includes the following information:
Fresh Air Temp
Relative Humidity
Internal Temperature
Set Point
HVAC Mode
Events/Faults
Etc, (full list below)*
This information is recorded about once every 30 seconds, and the logs carry about 7 days of information before being overwritten. So you end up with a lot of data, about 46,000 individual data points per Set per Day.
 
The event recorder records data about twice a second and you can get:
GPS location,
patronage levels (in kg),
direction,
Run #
door status
etc.
All this will influence HVAC power consumption.  The event recorder data is stored on the set for about 14 days before being overwritten.
 
Every 10 minutes a huge number of signals off the train is dumped to the shore (what’s in the ER plus other diagnostic information from many systems), this is stored in the Databank and then can be accessed by through Train DNA. Accessing this data can be a bit hit and miss as sometimes large chunks are missing. |

| 003 | 26MAY | A list of the metering instruments used to create each data type - including their calibration and testing schedules where possible. | For the HVAC Hardware Trial we used specific power meters and data loggers which were calibrated as we needed to gain a higher resolution compared to what was available using the existing on train power meters.
 
The on board energy meters do have calibration requirements however TLS does not complete these activities.
 
The HVAC units ‘metering instruments’ i.e. temperature and humidity sensors, don’t go through a calibration or testing schedule, rather they are maintained along according to the Technical Maintenance Plan (TMP), or fixed on failure. |
